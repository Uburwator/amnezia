# AmneziaWG Docker Установщик

Простые в использовании скрипты для установки VPN сервера AmneziaWG с использованием Docker, stateless контейнеров и конфигурацией на хосте.

## ⚠️ ПРЕДУПРЕЖДЕНИЕ О БЕЗОПАСНОСТИ

**НИКОГДА не скачивайте и не запускайте bash скрипты с правами root без проверки!**

Это чрезвычайно опасно и может скомпрометировать всю вашу систему. Перед запуском ЛЮБОГО скрипта из интернета:

1. **Клонируйте репозиторий** и проверьте ВЕСЬ исходный код
2. **Понимайте что делает каждая команда**
3. **Убедитесь что скрипты из доверенного источника**
4. **Проверьте отзывы сообщества и недавнюю активность**
5. **НИКОГДА не используйте pipe wget/curl в bash** (`curl | sudo bash` опасно!)

**Рекомендуемый процесс установки:**


## Возможности

- 🐋 **На базе Docker** - Работает в контейнерах, изолирован от хоста
- 💾 **Stateless контейнеры** - Вся конфигурация хранится на хосте
- 🔄 **Горячая перезагрузка** - Добавление клиентов без перезапуска сервера
- 🌐 **Поддержка DNS** - Используйте доменные имена вместо IP
- 🎭 **Обфускация трафика** - HTTP/DNS сигнатуры для обхода DPI
- 🔒 **Безопасность** - Постквантовая защита с preshared ключами
- 🚀 **Быстро** - Основан на протоколе WireGuard
- 🔧 **Multi-VPN** - Работает вместе с другими VPN серверами

## Установка

### Метод 1: Установка одной командой (Рекомендуется)

```bash
wget -O - https://raw.githubusercontent.com/YOUR-USERNAME/amneziawg-docker-install/main/install.sh | sudo bash
```

### Метод 2: Ручная установка

```bash
# Клонировать репозиторий
git clone https://github.com/YOUR-USERNAME/amneziawg-docker-install.git
cd amneziawg-docker-install

# Запустить установку
sudo bash scripts/setup.sh
```

### Опции конфигурации

Установите переменные окружения перед установкой:

```bash
export AWG_SUBNET_IP="10.66.66.1"    # IP сервера VPN (по умолчанию: 10.66.66.1)
export AWG_SUBNET_CIDR="24"          # Маска подсети (по умолчанию: 24)
export AWG_PORT="55555"               # Порт VPN (по умолчанию: 51820)
export SERVER_ENDPOINT="vpn.example.com"  # DNS имя сервера (автоопределение)

sudo -E bash install.sh
```

## Использование

### Добавить нового клиента

```bash
# С DNS именем
SERVER_ENDPOINT=vpn.example.com sudo -E amneziawg add-client laptop

# С IP адресом
SERVER_ENDPOINT=203.0.113.5 sudo -E amneziawg add-client phone

# Автоопределение IP сервера
sudo amneziawg add-client tablet
```

Конфигурация клиента будет сохранена в `/opt/amnezia/awg/clients/<имя>.conf`

### Управление сервером

```bash
# Показать статус и активные подключения
sudo amneziawg status

# Список всех клиентов
sudo amneziawg list

# Показать конфигурацию сервера
sudo amneziawg show

# Просмотр логов контейнера
sudo amneziawg logs

# Перезапуск/остановка/запуск
sudo amneziawg restart
sudo amneziawg stop
sudo amneziawg start

# Резервное копирование
sudo amneziawg backup
```

### Импорт конфига на клиент

**На iOS/Android:**
1. Скопируйте файл `.conf` на устройство
2. Откройте приложение AmneziaVPN
3. Настройки → Импорт из файла
4. Выберите файл конфигурации
5. Подключитесь!

**На Linux/macOS:**
```bash
# Установите инструменты AmneziaWG
# См.: https://github.com/amnezia-vpn/amneziawg-tools

# Подключение
sudo awg-quick up laptop.conf
```

### Процесс установки

1. **Создание директорий**: `/opt/amnezia/awg/` для конфигурации
2. **Генерация ключей**: Приватные/публичные ключи сервера, preshared ключ
3. **Генерация параметров обфускации**: Случайные значения Jc, S1-S4, H1-H4
4. **Создание Docker сети**: `amnezia-dns-net` для опционального DNS контейнера
5. **Запуск контейнера**: Монтирование директории конфига, публикация VPN порта
6. **Настройка firewall**: iptables правила внутри контейнера для NAT

### Процесс добавления клиента

1. **Генерация ключей клиента**: Уникальная пара приватный/публичный ключ
2. **Назначение IP**: Следующий доступный IP в VPN подсети
3. **Добавление peer**: Добавление в конфиг сервера
4. **Перезагрузка сервера**: Горячая перезагрузка без рестарта
5. **Создание конфига клиента**: Со всеми параметрами обфускации
6. **Добавление сигнатур**: HTTP/DNS имитация трафика (I1-I5)

## Обфускация трафика

AmneziaWG маскирует VPN трафик под легитимные HTTP/DNS запросы используя:

### Мусорные пакеты (Jc, Jmin, Jmax)
Случайные пакеты разного размера, отправляемые перед handshake для непредсказуемости трафика.

### Дополнение (S1-S4)
- **S1**: Дополнение инициации handshake
- **S2**: Дополнение ответа handshake
- **S3**: Дополнение cookie reply
- **S4**: Дополнение транспортных сообщений

### Магические заголовки (H1-H4)
Рандомизация поля типа пакета с неперекрывающимися диапазонами для избежания характерных сигнатур WireGuard.

### Сигнатурные пакеты (I1-I5)
Пользовательские пакеты, отправляемые перед handshake, имитирующие реальные протоколы:
- **I1**: `GET / HTTP/1.1\r\nHost: ` - Начало HTTP запроса
- **I2**: Случайное доменное имя + `.com\r\n`
- **I3**: `User-Agent: Mozilla/5.0\r\n` - Заголовок браузера
- **I4**: `Connection: keep-alive\r\n\r\n` + случайные данные
- **I5**: DNS запрос с временной меткой

## Требования к Firewall

### Внешний Firewall (у провайдера облака)

**Обязательно**: Разрешить UDP порт (по умолчанию: 51820) входящий

Настройте в панели управления провайдера:
- **DigitalOcean**: Networking → Firewalls
- **Hetzner**: Firewalls → Rules
- **Vultr**: Firewall → Add Rule
- **AWS/Lightsail**: Networking → Firewall

### Firewall контейнера (Автоматически)

Настраивается автоматически через `start.sh`:
- Включена IP переадресация
- NAT/masquerading для VPN подсети
- Правила forwarding для интерфейса awg0

**Ручная настройка не требуется!**

## Решение проблем

### Клиент подключается, но нет интернета

**Симптомы**: VPN показывает подключено, но интернет не работает

**Причина**: Отсутствуют или неверны iptables правила контейнера

**Решение**:
```bash
# Проверьте, есть ли правила iptables в start.sh
cat /opt/amnezia/awg/start.sh | grep iptables

# Если отсутствуют, контейнеру нужны NAT правила
# Перезапустите контейнер:
docker restart amnezia-awg

# Проверьте NAT:
docker exec amnezia-awg iptables -t nat -L POSTROUTING -n
# Должны быть MASQUERADE правила для VPN подсети
```

### Handshake не происходит

**Симптомы**: Нет handshake в `awg show`, клиент постоянно повторяет попытки

**Причина**: Несовпадение конфигураций клиента и сервера

**Решение**:
```bash
# Проверьте публичный ключ клиента в конфиге сервера
docker exec amnezia-awg awg show awg0
# Сравните публичный ключ peer с конфигом клиента

# Проверьте совпадение параметров обфускации
grep "^Jc = " /opt/amnezia/awg/awg0.conf
grep "^Jc = " /opt/amnezia/awg/clients/laptop.conf
# Все параметры должны быть идентичны!
```

### Ошибка 900: ImportInvalidConfigError

**Симптомы**: Приложение AmneziaVPN показывает ошибку при импорте конфига

**Причины и решения**:

1. **Пустые параметры обфускации**
   - Проверьте, что конфиг имеет значения: `Jc = 5` а не `Jc = `
   - Перегенерируйте клиента обновленным скриптом

2. **Отсутствуют параметры I1-I5**
   - Старые конфиги без сигнатурных пакетов
   - Используйте последний скрипт add-client

3. **Лишние пробелы или неверный формат**
   - Проверьте: `cat -A /opt/amnezia/awg/clients/laptop.conf`
   - Не должно быть `$` в конце строк (кроме реального конца строки)

4. **Использование QR кода вместо файла**
   - **Решение**: Используйте импорт файла, НЕ QR код
   - iOS приложение имеет проблемы со сжатым QR форматом

### QR код не импортируется

**Симптомы**: `qUncompress: Input data is corrupted`

**Причина**: iOS приложение AmneziaVPN имеет проблемы с Qt форматом сжатия

**Решение**: **Используйте импорт файла**
- Импорт файла работает надежно
- QR коды экспериментальные и багованные
- Передавайте `.conf` файл через AirDrop/iCloud/Email

### Порт уже используется

**Симптомы**: Контейнер не запускается, конфликт портов

**Решение**:
```bash
# Проверьте что использует порт
sudo netstat -uln | grep 55555
# или
sudo ss -uln | grep 55555

# Измените порт при необходимости
export AWG_PORT="51821"
sudo -E bash install.sh
```

### Контейнер не может достучаться до интернета

**Симптомы**: `docker exec amnezia-awg ping 8.8.8.8` не работает

**Причина**: Docker networking настроен неверно

**Решение**:
```bash
# Проверьте маршрут по умолчанию контейнера
docker exec amnezia-awg ip route

# Должно показать:
# default via X.X.X.X dev eth0

# Если отсутствует - сеть контейнера сломана
# Пересоздайте контейнер или проверьте установку Docker
```

### Несколько Docker сетей

**Симптомы**: Контейнер имеет eth0 и eth1, маршрутизация неверна

**Причина**: Контейнер подключен к нескольким Docker сетям

**Решение**:
```bash
# Проверьте маршруты контейнера
docker exec amnezia-awg ip route

# Если default via eth0, но интернет на eth1:
# start.sh уже это обрабатывает - NAT правила для ОБОИХ интерфейсов

# Проверьте:
docker exec amnezia-awg iptables -t nat -L -n | grep MASQUERADE
# Должны быть правила для eth0 и eth1
```

### Клиенты не могут связаться друг с другом

**Симптомы**: Клиент 1 не может пинговать Клиент 2

**Причина**: Forwarding между VPN клиентами отключен

**Решение**:
Добавьте в `/opt/amnezia/awg/start.sh`:
```bash
iptables -A FORWARD -i awg0 -o awg0 -j ACCEPT
```

Затем перезапустите контейнер.

## Продвинутая конфигурация

### Изменить DNS серверы

Отредактируйте конфиги клиентов для использования других DNS:

```bash
# Использовать Google DNS
DNS = 8.8.8.8, 8.8.4.4

# Использовать Quad9
DNS = 9.9.9.9, 149.112.112.112

# Использовать свой DNS сервер на хосте
DNS = 10.66.66.1
```

### Настроить обфускацию

Отредактируйте `/opt/amnezia/awg/awg0.conf` перед первым клиентом:

```ini
Jc = 7        # Больше мусорных пакетов (медленнее но скрытнее)
Jmin = 100    # Больший минимальный размер мусора
Jmax = 1280   # Максимальный размер (предел MTU)
```

**Важно**: Все клиенты должны быть перегенерированы после изменения этих значений!

### Отключить сигнатурные пакеты

Если сигнатурные пакеты вызывают проблемы, отредактируйте скрипт add-client и установите:

```bash
I1=''
I2=''
I3=''
I4=''
I5=''
```

Или полностью удалите эти строки из конфигов клиентов.

### Резервное копирование


```bash

# Перезапустить контейнер
docker restart amnezia-awg
```

   - Нативный WG: `10.0.0.0/24`

   - Нативный WG: `51820`

## Безопасность

### Какие данные хранятся

**На VPS**:
- Приватный ключ сервера (`/opt/amnezia/awg/server_private.key`)
- Приватные ключи клиентов (`/opt/amnezia/awg/clients/*.conf`)
- Preshared ключ (общий для всех клиентов)

**Лучшие практики**:
1. Установите правильные права доступа (делается автоматически)
2. Шифруйте резервные копии перед удаленным хранением
3. Используйте сильные пароли для доступа к VPS
4. Включите SSH аутентификацию по ключам
5. Обновляйте Docker и систему хоста

### Приватность

**Нет телеметрии**: Этот установщик не отправляет данные третьим лицам

**Анализ трафика**: Параметры обфускации затрудняют DPI обнаружение, но не делают его невозможным

**Утечки DNS**: Настройте клиентов использовать VPN DNS для предотвращения утечек

## FAQ

### В: Могу ли я использовать это с существующим WireGuard?
**О**: Да! Используйте разные подсети и порты. Оба могут работать одновременно.

### В: Нужно ли перезапускать контейнер при добавлении клиентов?
**О**: Нет! Скрипт использует горячую перезагрузку (`awg syncconf`).

### В: Могу ли я изменить IP сервера позже?
**О**: Да, если используете DNS имена. Просто обновите DNS записи. Если используете IP адреса, перегенерируйте все конфиги клиентов.

### В: В чем разница от стандартного WireGuard?
**О**: AmneziaWG добавляет обфускацию для обхода DPI. Протокол совместим, но включает дополнительные параметры.

### В: Это безопасно?
**О**: Да. Основан на WireGuard (проверенный аудитом), добавляет обфускацию и preshared ключи (постквантовая безопасность).

### В: Почему не использовать официальный клиент Amnezia?
**О**: Это дает полный контроль, работает на серверах без GUI, и хранит конфиг на VPS для легкого управления.

### В: QR коды не работают?
**О**: Известная проблема с iOS приложением и сжатием. Используйте импорт файла - это надежнее.

### В: Могу ли я запустить это без Docker?
**О**: Используйте нативный установщик: https://github.com/wiresock/amneziawg-install
